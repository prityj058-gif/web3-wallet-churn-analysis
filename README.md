 # 🚀 Web3 Wallet Churn Analysis

## 📌 Project Overview

This project analyzes blockchain wallet transaction behavior using Python-based data analytics techniques. The goal is to understand user engagement, wallet activity patterns, transaction trends, and potential churn behavior within a Web3 ecosystem.

The project demonstrates practical skills in:
- Data Cleaning
- Exploratory Data Analysis (EDA)
- Data Visualization
- User Segmentation
- Behavioral Analytics

---

# 🎯 Project Objectives

The main objectives of this project were:

- Analyze cryptocurrency transaction data
- Understand wallet activity patterns
- Identify active and inactive users
- Segment users using RFM analysis
- Detect potential churn behavior
- Generate business insights from blockchain data

## ✔ RFM Analysis

Users were segmented based on:

- **Recency** → How recently a wallet was active
- **Frequency** → How often transactions occurred
- **Monetary** → Total transaction value

This helped identify:
- Loyal users
- High-value wallets
- Dormant users
- Churn-risk users

---

# 📈 Key Findings

## 🔍 Wallet Behavior Insights

- A small percentage of wallets contributed a large portion of transaction value.
- Many wallets showed irregular transaction frequency.
- Some users became inactive after periods of high activity.

---

## 🔍 Gas Fee Insights

- Higher gas fees were associated with reduced transaction frequency.
- Peak transaction periods often showed increased gas usage.

---

## 🔍 User Segmentation Insights

RFM analysis revealed:
- Highly active wallets
- Medium-engagement users
- Low-activity users
- Potential churn-risk segments

- Early churn detection
